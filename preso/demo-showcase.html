The Brief
Intelligent Order Orchestration
A Fortune 500 retailer needed real-time order orchestration across 2,000 stores with sub-10ms latency, multi-step fulfillment workflows, and zero data loss — during peak holiday traffic of 10,000 concurrent orders per second.
  • Real-time inventory reservation across 2,000 store locations with conflict resolution
  • Multi-step order fulfillment workflows with automatic compensation and rollback on failure
  • Event-driven notifications to customers, warehouse systems, and shipping partners
  • Sub-10ms p99 response times at 10,000 concurrent orders per second
  • Full audit trail of every state change for compliance and dispute resolution
  • Multi-region active-active deployment for zero-downtime during regional failures

Try It Yourself
Run this on your machine in 10 minutes
Everything you just saw is yours to explore. Clone the repo, start the runtime, and interact with a production-grade distributed system running locally. No cloud account needed.
01
Install an AI coding assistant
Pick your tool. Akka Specify works with any AI assistant that supports plugins or MCP.
$ npm install -g @anthropic-ai/claude-code
VS Code > Extensions > search "GitHub Copilot" > Install
$ npm install -g @anthropic-ai/gemini-cli
Download from windsurf.com > Install > Launch
02
Install Akka Specify
Get the Akka Specify plugin into your AI tool. Choose your method:
For Claude Code and any AI tool with plugin marketplace support. Other tools will follow this pattern as plugin support expands.
> /plugin marketplace add akka/ai-marketplace
> /plugin install akka@ai-marketplace
> /reload-plugins
Akka Specify skills installed
For any MCP-compatible AI tool. Clone the Specify plugin repo to get the MCP config, then open your assistant from the project directory.
$ akka specify init my-project
$ cd my-project
# Project includes _mcp.json for auto-registration
# Open your AI assistant from this directory
doc.akka.io/sdk/spec-driven-development.html
