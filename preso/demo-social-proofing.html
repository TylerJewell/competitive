The Brief
Social Proofing Agent
An e-commerce retailer needed AI-powered social proof messages on product pages — "Only 3 left!" or "Bought 12 times in the last hour" — generated intelligently from real-time view counts, order signals, and inventory levels, with sub-10ms cached reads at scale.
  • Ingest pre-aggregated view counts and order events from external analytics
  • AI agent selects optimal strategy (URGENCY, VALIDATION, SCARCITY, TRENDING) based on signals
  • Generate natural, category-aware copy under 60 characters
  • Cache messages in entity state — zero LLM calls on the read path
  • Real-time SSE streaming for live UI updates
  • Suppress messages when activity falls below threshold

(6)Domain Objects
Strategy Enum Social proof strategy types
Defines the four social proof strategies the agent can select based on real-time product signals.
Strategy.java
// Agent selects one strategy per evaluation cycle public enum Strategy { /** Low inventory — "Only 3 left in stock!" */ SCARCITY, /** High recent orders — "Bought 47 times today" */ VALIDATION, /** Rapid velocity increase — "Trending now!" */ TRENDING, /** Time-limited signal — "Selling fast, don't miss out" */ URGENCY }
TrendDirection Enum Signal velocity direction
Indicates whether product activity is accelerating, stable, or declining. Used by the agent to decide if a TRENDING strategy is appropriate.
TrendDirection.java
public enum TrendDirection { RISING, STABLE, FALLING }
ProductSignals Record Aggregated activity signals per product
Pre-aggregated metrics from external analytics. The agent uses these signals to pick a strategy and generate contextual copy.
ProductSignals.java
public record ProductSignals( int viewsLastHour, int ordersLastHour, int inventoryRemaining, TrendDirection trend, String category, Instant collectedAt ) { public boolean meetsThreshold() { return viewsLastHour > 5 || ordersLastHour > 0; } }
ProductState Record Entity state with cached social proof
The entity's persisted state. Holds the latest signals and the cached social proof message, enabling zero-LLM reads.
ProductState.java
public record ProductState( String productId, ProductSignals latestSignals, CachedSocialProof cachedProof, Instant lastUpdated ) { public static ProductState empty(String id) { return new ProductState(id, null, null, Instant.EPOCH); } public boolean hasActiveProof() { return cachedProof != null && !cachedProof.isExpired(); } }
ProductEvent Sealed Interface 5 event types for product lifecycle
Sealed event hierarchy ensures compiler-enforced exhaustive handling across all consumers and event handlers.
ProductEvent.java
public sealed interface ProductEvent { record SignalsAggregated( ProductSignals signals ) implements ProductEvent {} record ProofGenerated( Strategy strategy, String message, Instant generatedAt ) implements ProductEvent {} record ProofCached( CachedSocialProof proof ) implements ProductEvent {} record ProofSuppressed( String reason ) implements ProductEvent {} record ProofExpired( String productId ) implements ProductEvent {} }
CachedSocialProof Record Cached message with TTL and strategy
The generated social proof message stored in entity state. Includes the strategy used, the copy text, and expiration for automatic refresh.
CachedSocialProof.java
public record CachedSocialProof( Strategy strategy, String message, Instant generatedAt, Instant expiresAt ) { public boolean isExpired() { return Instant.now().isAfter(expiresAt); } public static CachedSocialProof create( Strategy s, String msg, Duration ttl) { var now = Instant.now(); return new CachedSocialProof( s, msg, now, now.plus(ttl)); } }
(1)Event Sourced Entities
ProductEntity ESE 5 events, 6 commands
Core entity keyed by productId. Ingests aggregated signals, stores cached social proof messages, and serves reads with zero LLM calls. Full audit trail of every signal and proof generation.
ProductEntity.java
@ComponentId("product") public class ProductEntity extends EventSourcedEntity<ProductState, ProductEvent> { @CommandHandler public Effect<String> aggregateSignals( ProductSignals signals) { return effects() .persistEvent(new SignalsAggregated(signals)) .thenReply(__ -> "OK"); } @CommandHandler public Effect<String> cacheProof( CachedSocialProof proof) { return effects() .persistEvent(new ProofCached(proof)) .thenReply(__ -> "OK"); } @CommandHandler public readonly Effect<CachedSocialProof> getProof() { var state = currentState(); if (state.hasActiveProof()) return effects().reply(state.cachedProof()); return effects().error("No active proof"); } @EventHandler public ProductState onSignalsAggregated( SignalsAggregated evt) { return new ProductState( commandContext().entityId(), evt.signals(), currentState().cachedProof(), Instant.now()); } @EventHandler public ProductState onProofCached( ProofCached evt) { return new ProductState( currentState().productId(), currentState().latestSignals(), evt.proof(), Instant.now()); } }
(1)Agents
SocialProofAgent Agent Strategy selection + copy generation
AI agent that evaluates product signals, selects the optimal social proof strategy (URGENCY, VALIDATION, SCARCITY, TRENDING), and generates natural category-aware copy under 60 characters. Only invoked on signal changes — never on reads.
SocialProofAgent.java
@ComponentId("social-proof-agent") public class SocialProofAgent extends Agent { private final ComponentClient client; public Effect<CachedSocialProof> evaluate( String productId, ProductSignals signals) { if (!signals.meetsThreshold()) { return effects().reply(null); } var strategy = selectStrategy(signals); var prompt = buildPrompt( strategy, signals); return effects() .asyncCall(() -> llm().generate(prompt)) .thenApply(copy -> { var proof = CachedSocialProof .create(strategy, copy, Duration.ofMinutes(15)); client.forEventSourcedEntity(productId) .method(ProductEntity::cacheProof) .invokeAsync(proof); return proof; }); } private Strategy selectStrategy( ProductSignals s) { if (s.inventoryRemaining() < 5) return Strategy.SCARCITY; if (s.trend() == RISING) return Strategy.TRENDING; if (s.ordersLastHour() > 20) return Strategy.VALIDATION; return Strategy.URGENCY; } }
(1)Consumers
SocialProofConsumer Consumer Reacts to SignalsAggregated
Listens for SignalsAggregated events from ProductEntity and invokes the SocialProofAgent to evaluate signals and generate new social proof copy. Bridges the event stream to the AI pipeline.
SocialProofConsumer.java
@ComponentId("social-proof-consumer") public class SocialProofConsumer extends Consumer { private final ComponentClient client; @Consume.FromEventSourcedEntity( ProductEntity.class) public Effect onSignalsAggregated( ProductEvent.SignalsAggregated evt) { var productId = messageContext().eventSubject().get(); return effects().asyncEffect( client.forAgent() .method(SocialProofAgent::evaluate) .invokeAsync(productId, evt.signals()) .thenApply(__ -> effects().done())); } }
(1)Views
SocialProofView View Projections + SSE streaming
Read-optimized projection that consumes ProofCached events and serves them via SSE for real-time UI updates. Supports query by productId and category-level aggregations.
SocialProofView.java
@ComponentId("social-proof-view") public class SocialProofView extends View { public record ProofRow( String productId, String message, Strategy strategy, Instant generatedAt, Instant expiresAt ) {} @Consume.FromEventSourcedEntity( ProductEntity.class) public UpdateEffect<ProofRow> onProofCached( ProductEvent.ProofCached evt) { var p = evt.proof(); return effects().updateRow( new ProofRow( updateContext().eventSubject().get(), p.message(), p.strategy(), p.generatedAt(), p.expiresAt())); } @Query("SELECT * FROM social_proofs WHERE productId = :id") public QueryEffect<ProofRow> getByProduct( String id) { return queryResult(); } }
(4)Endpoints
DemoUIEndpoint Endpoint Serves the live demo HTML page
Serves the interactive demo UI that displays product cards with live social proof messages. Connects to SSE endpoint for real-time updates.
DemoUIEndpoint.java
@HttpEndpoint("/") public class DemoUIEndpoint { @Get("/") public HttpResponse index() { return HttpResponses.ok() .withContentType("text/html") .withBody(renderDemoPage()); } }
ProductEndpoint Endpoint Product entity CRUD + proof reads
HTTP API for reading product state and cached social proof messages. Sub-10ms reads from entity state with no LLM involvement.
ProductEndpoint.java
@HttpEndpoint("/api/products") public class ProductEndpoint { private final ComponentClient client; @Get("/{productId}/proof") public Effect<CachedSocialProof> getProof( String productId) { return client .forEventSourcedEntity(productId) .method(ProductEntity::getProof) .invokeAsync(); } @Get("/{productId}/state") public Effect<ProductState> getState( String productId) { return client .forEventSourcedEntity(productId) .method(ProductEntity::getState) .invokeAsync(); } }
IngestionEndpoint Endpoint Signal ingestion from analytics
Accepts pre-aggregated view counts and order events from external analytics pipelines. Routes signals to the appropriate ProductEntity by productId.
IngestionEndpoint.java
@HttpEndpoint("/api/ingest") public class IngestionEndpoint { private final ComponentClient client; @Post("/{productId}/signals") public Effect<String> ingestSignals( String productId, ProductSignals signals) { return client .forEventSourcedEntity(productId) .method(ProductEntity::aggregateSignals) .invokeAsync(signals); } }
SocialProofEndpoint Endpoint SSE streaming + view queries
Exposes SSE streaming for real-time social proof updates and query endpoints for bulk retrieval. Powers the live demo UI.
SocialProofEndpoint.java
@HttpEndpoint("/api/social-proof") public class SocialProofEndpoint { private final ComponentClient client; @Get("/stream") public Effect<ServerSentEvents> stream() { return client.forView() .method(SocialProofView::streamAll) .invokeServerSentEvents(); } @Get("/{productId}") public Effect<ProofRow> getByProduct( String productId) { return client.forView() .method(SocialProofView::getByProduct) .invokeAsync(productId); } }
