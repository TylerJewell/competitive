# Akka Sales Presentation

Maintainable source tree for the Akka sales presentation. Every slide is a separate
file. The builder assembles them into a complete HTML file with all transitions intact.

## Quick Start

```bash
cd sales-presentation

# Build the standard overview deck
python3 builder/build.py

# Build a personalized deck for a specific presenter
python3 builder/build.py --presenter tyler

# Build shareable (offline screenshots) variant
python3 builder/build.py --mode shareable --presenter tyler

# Output is always: generated/<mode>/index.html
```

## Directory Layout

```
sales-presentation/
├── slides/              # One folder per slide
│   ├── 00-title/
│   │   ├── slide.css    # Section-scoped CSS
│   │   ├── slide.html   # HTML fragment (no <html>/<head>/<body>)
│   │   ├── slide.js     # Section-scoped JS
│   │   └── meta.json    # Spacers, scroll envelopes, reveal thresholds, images
│   └── ... (01-hero through 11-closing)
│
├── shell/               # Scaffold files (generated by migrate.py — do not edit)
│   ├── shell.html       # HTML template with {{SLIDES_*}} placeholders
│   ├── shared.css       # CSS reset, :root vars, keyframes
│   ├── nav.js           # Keyboard navigation JS
│   └── kiosk.js         # Kiosk auto-advance JS (?kiosk=<seconds> URL param)
│
├── builder/
│   ├── build.py         # Assembles slides → generated/
│   ├── migrate.py       # One-shot extraction from preso/templates/base.html
│   ├── slide-registry.json  # Ordered slide list (read by build.py)
│   └── image-registry.json  # All images referenced across slides
│
├── assets/
│   ├── images/          # All presentation images (PNG/JPG)
│   └── logos/           # Customer and partner logos
│
├── presenters/
│   └── tyler.json       # Tyler's name, title, LinkedIn URL
│
├── demos/
│   ├── _plugin/         # /akka:demo Claude skill
│   │   ├── demo.md      # Skill command file
│   │   └── SPEC.md      # Full skill specification
│   └── social-proofing/ # Social-proofing demo reference
│
└── generated/           # Build outputs (git-ignored)
    └── overview/
        └── index.html
```

## Editing a Slide

1. Edit the relevant file in `slides/XX-name/`
2. Run `python3 builder/build.py` to regenerate
3. Open `generated/overview/index.html` in a browser to preview

## Updating the Governance Section

The governance section (slide 07) has its own source at
`../sections/section-7-governance.html`. After editing that file:

```bash
# 1. Splice updated section into base template
python3 ../preso/splice-section7.py
# → writes preso/templates/base-draft.html

# 2. Re-extract all slides from the draft
python3 builder/migrate.py --source base-draft.html

# 3. Rebuild
python3 builder/build.py
```

## Adding a New Presenter

Create `presenters/<name>.json`:
```json
{
  "name": "Jane Smith",
  "title": "Account Executive, Akka",
  "linkedin": "https://www.linkedin.com/in/janesmith/",
  "slug": "jane"
}
```

Then build with `--presenter jane`.

## Build Modes

| Mode | Description |
|------|-------------|
| `overview` | Generic pitch, no presenter name (default) |
| `shareable` | Static screenshots, works fully offline |
| `live` | Embedded running app (iframe) |
| `hands-on` | Includes setup guide for recipient to run it |

Slides opt in/out of modes via `meta.json → include_in`.
